API

Application Programming Interface
It is a set of rules that allows programs to talk to each other.
The developers create the api on the server side and allows the client to talk with it.

API: superset
Rest API: subset


REST API
A set of constraints to be used for creating web services.

Example 
Client-server architecture: 
Stateless: server cant store any information about client.
Cacheble: Data can be cached in order to increase performance.
Layered Architecture.

HTTP methods for clients:
GET: Read resource
PUT: Updates an existing resource
POST: It creates new resource
DELETE: It deletes resources

404: Resource not found
200: success
201: created
401: unauthorized
500: Server Error
